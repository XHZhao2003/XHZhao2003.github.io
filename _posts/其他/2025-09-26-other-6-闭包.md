### Python 命名空间和作用域

命名空间（namespace）是从一个名字到对象的映射，通常使用字典来实现。Python 中的赋值就是将一个变量名绑定到一个对象上，使用关键字 `del` 删除一个变量就是解除一个绑定。那么所有绑定的记录就是命名空间。

为了避免命名冲突，有不同的命名空间

- 内置命名空间（Built-in Namespace）：例如 `len`, `print` 等 Python 内置函数

- 全局命名空间（Global Namespace）：一个模块顶层定义的变量

- 局部命名空间（Local Namespace）：一个函数内部定义的变量

- 类命名空间（Class Namesapce）：类定义中的属性和方法

```Python
x = 10  # Global Namespace

def func():
    y = 20  # Local Namespace
    print(x)  # Built-in Namespace
```

一个变量的作用域（Scope）是程序中的一个文本区域，在这个区域中，这个变量可以通过某个命名空间绑定到对象上；在这个区域之外，则无法通过任何命名空间进行绑定。

作用域有以下几种

- 局部作用域 Local Scope：函数内部定义的变量，可以在本函数内部被访问到。

    ```Python
    def func():
        x = 10  
        print(x) # x in Local Scope
    ```

- 闭包作用域 Enclosing Scope：对于嵌套函数而言，外部函数中定义的变量可以被内部函数访问到。"外部函数"可以经过多层嵌套。

    ```Python
    def outer():
        x = 20
        def inner():
            print(x)  
            # x in Enclosing Scope
            # using Local Namespace of outer function
        inner()

    ```

- 全局作用域 Global Scope：在模块顶层定义的变量，可以在模块内的任意位置被访问到。

    ```Python
    x = 30
    def func():
        print(x)  # x in Global Scope, using Global Namespace
    func()
    ```

- 内置作用域 Built-in Scope：Python 内置的名称，可以在任意位置被访问到。


正是由于命名冲突，可能出现的情况是，多个相同名称的变量的作用域重叠在一起。这种情况下，为了得到唯一无歧义的变量应用，Python 使用 LEGB 原则。也就是，首先查看该变量是否存在于 Local Scope 中，如果不存在，就依次查看是否存在在 Enclosing Scope, Global Scope, Built-in Scope 中。

```Python
x = 'x from global'  # Global
def outer():
    x = 'x from outer'
    def inner():
        print(x)    # x in Enclosing Scope
    inner()
    print(x)    # x in Local Scope
outer()
print(x) # x in Global Scope
```

需要注意的是，在读取变量时，Python 的默认行为是使用 LEGB 原则逐层查找引用。但是在写入变量时，Python 的默认行为是在当前局部作用域内创建新变量，不会按照 LEGB 规则向外查找。

```Python
x = "global"
def f():
    x = "local"  # Creating new local variable
    print(x)     # x in Local Scope
f()
print(x)         # x in Global Scope, still 'global', not modified
```

Python 的 `global` 关键字可以改变上述默认行为。在函数内，如果一个变量被声明为 global，那么对这个变量的读写都将作用于 Global Namespace，这个变量也不会被视为局部变量。

```Python
x = "global"
def f():
    global x
    print(x)          # Read x, in Global Scope, using Global Namespace
    x = "modified"    # Write x, in Global Scope. Create x if not existing
f()
print(x)              # x becomes "modified"
```

Python 中的 `nonlocal` 关键字也可以改变默认行为。在嵌套函数中，如果将内部函数中的变量声明为 nonlocal，那么对这个变量的读写将作用于 Enclosing Scope，也就是最近一层的外部函数。

```Python
def outer():
    x = "enclosing"
    def inner():
        nonlocal x     # Error if x not in Enclosing Scope
        print(x)       # Read x, in Enclosing Scope
        x = "changed"  # Write x, in Enclosing Scope
    inner()
    print(x)           # "changed"

outer()
```

nonlocal 和 global 的一个区别是：声明 `global x` 时，x 可以不存在于全局作用域，因为它的作用只是说明，对 x 的引用应该使用全局命名空间。而声明 `nonlocal x` 时，x 必须存在于闭包作用域，因为正在尝试将这个 x 与闭包作用域中的变量 x 进行绑定。


### 闭包 Enclosure

闭包函数是嵌套的内部函数，它能够访问其定义时所在的作用域中的变量，即使这个函数在这个作用域之外被调用。

这个定义过于抽象，所以还是用一个实际的例子来解释什么是闭包，为什么使用闭包，以及如何使用闭包。

闭包的一个经典使用场景是，实现带有状态的函数。例如，考虑一个平面坐标上的棋盘游戏，棋子最初的位置是 (0. 0)，我们希望函数每次接收一组参数 (dx, dy)，然后返回移动后的位置。也就是说，函数还需要能够记住上一步走到了哪里。

不考虑闭包，我们可以用一个类实现上面的功能，很简单。但由于这只是一个演示的例子，我们仍然选择用闭包来实现这个功能。

```Python
def create():
    pos = [0,0]

    def go(direction):
        new_x = pos[0] + direction[0]
        new_y = pos[1] + direction[1]
        
        pos[0] = new_x
        pos[1] = new_y
        
        return pos    
    return go

player = create()
print(player([1,0])) # [1, 0]
print(player([0,1])) # [1, 1]
```

这个例子中，闭包函数是内部函数 go 函数。它处于 pos 的 Enclosing Scope，并且在内部引用了 pos 对象，具体来说就是调用了 pos 的 setitem 和 getitem 方法。并且，外部函数 create 将内部函数 go 作为返回值赋值给了变量 player。当我们运行 `player = create()` 的时候，就产生了一个闭包函数。也就是说，player 指向的这个 go 函数能够访问外部函数中的 pos，即使调用的位置已经离开了外部函数的 Local Scope。

player 是一个函数对象，具有 `__closure__` 属性，这个属性是一个由 cell 对象组成的列表，每一个 cell 对象表示了它在外部函数中引用的变量。在这个例子中，只有一个变量，也就是 pos，它是 `player.__closure__[0]`。

下面这个李忠，我们希望实现一个打印函数（借助 Python 内置的 print 函数），用来打印字符串。同时，这个函数还需要输出当前是第几次打印。


```Python
def make_print():
    count = 0  # 外层函数的局部变量，用于记录调用次数

    def print_with_count(message):
        nonlocal count  # 声明要修改外层的 count
        count += 1
        print(f"[第 {count} 次] {message}")

    return print_with_count  # 返回闭包
```

这个例子的区别在于，闭包函数 print_with_count 希望使用外部函数的 count 变量。但是与可变的列表对象 pos 不同，count 对象是不可变的，对 count 的修改实际会进行一次重新绑定。所以，在闭包函数中就需要使用 nonlocal 关键字，表示对这个变量的绑定总是作用域 Enclosing Scope，具体来说也就是 funcion 对象的 `__closure__` 属性。


### 装饰器 Decorator

闭包的一类使用场景是装饰器 (Decorator)。装饰器接收一个函数，返回一个新函数。返回的新函数能够对原函数的功能进行增强。它的意义在于，不必要侵入式地修改原函数，从而实现更灵活地组装。这些增强的功能可能是：权限校验、计时器、输出日志，等等。

下面展示了一个最简单的装饰器

```Python
def my_decorator(func):
    def wrapper(*args, **kwargs):
        print("调用前：准备日志")
        result = func(*args, **kwargs)
        print("调用后：记录结束")
        return result
    return wrapper  #  返回闭包（wrapper 捕获了 func）

@my_decorator
def greet(name):
    print(f"Hello, {name}!")

greet("Alice")
```

这里，wrapper 是闭包函数。`@my_decorator` 是一个语法糖，由于它注解在 greet 函数上，它等价于 `greet = my_decorator(greet)`。换句话说，加上这个注释之后，greet 就被重新赋值为一个闭包函数了，它的闭包属性就包含原来的函数对象 greet。传入 greet 的参数（name）会以 `wrapper(*args, **kwargs)` 的形式传递给 wrapper 函数。而在 wrapper 函数内部，为了保证原有函数的功能不变，我们总是要执行原函数，如果有返回值那么最后也需要返回。在这基础上，我们可以加入自定义的功能，例如打印日志。

我们可以加入一些更复杂的功能。例如，我们可以在外部函数中维护一些状态（例如 LRU 缓存），并在 wrapper 函数中引用这些状态，从而实现原函数变得有状态。

我们还可以使用多层闭包，创造带参数的装饰器，例如下面这个例子：

```Python
def repeat(times):  # 装饰器工厂
    def decorator(func):  # 真正的装饰器
        def wrapper(*args, **kwargs):  # 闭包函数
            for _ in range(times):
                result = func(*args, **kwargs)
            return result
        return wrapper  # 返回闭包（捕获了 func 和 times）
    return decorator

@repeat(3)
def say_hello():
    print("Hello!")

say_hello()
```

在这个例子中，decorator 装饰器和前一个例子基本相同。它内部的闭包函数 wrapper 能够让原函数重复执行若干次。不同点在于，decorator 本身也是一个闭包函数。它的外部函数 repeat 是一个装饰器工厂，传入不同的参数，它返回不同的装饰器。

所以，当使用 `@repeat(3)` 进行装饰时，这个过程等价于 `say_hello = repeat(3)(say_hello)`。`repeat(3)` 返回一个装饰器，一个函数对象，它的闭包属性带有 times=3 这个属性。这个装饰器再接收 say_hello 函数，返回一个经过增强的 wrapper 函数。这一系列操作之后，say_hello 将具有两条闭包属性，第一个是原来的函数对象 say_hello，第二个则是来自 repeat 函数的参数 times。
